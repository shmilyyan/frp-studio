## 一、产品定位

| 项目 | 内容 |
|------|------|
| **产品名称** | FRP Studio |
| **目标用户** | 有内网穿透需求但不熟悉命令行的开发者、运维人员、个人用户 |
| **核心价值** | 将 frpc 客户端的配置、管理、监控全流程可视化；frps 服务端通过配套 shell 脚本一键部署，两端职责清晰，降低整体使用门槛 |

---

## 二、产品交付物构成

```
FRP Studio 完整交付
├── FRP Studio GUI
│   └── Windows / macOS 桌面客户端（frpc 管理）
└── frps-deploy 脚本包
    ├── install.sh        # 安装 frps + 注册系统服务
    ├── uninstall.sh      # 卸载清理
    ├── update.sh         # 版本升级
    └── frps.toml.tpl     # 配置模板
```

> frps 脚本包作为独立资源托管，GUI 内置下载入口与使用引导，用户复制一行命令到服务器即可完成部署。

**第 1 步：选择隧道类型**
```
  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
  │   TCP    │  │   UDP    │  │  HTTP    │  │  HTTPS   │

  └──────────┘  └──────────┘  └──────────┘  └──────────┘

  常用模板快速选择：
  [SSH 远程登录]  [Windows 远程桌面]  [本地网站]  [数据库连接]
```

**第 2 步：填写配置**
```
  隧道名称:    [ SSH-Home                    ]
  本地地址:    [ 127.0.0.1  ] 端口: [ 22   ]
  远程端口:    [ 6022       ]（服务器上映射的端口）
  所属节点:    [ my-server ▼]
```

**第 3 步：确认并启动**
```
  配置预览（TOML）:
  ┌─────────────────────────────┐
  │ [[proxies]]                 │
  │ name = "SSH-Home"           │
  │ type = "tcp"                │
  │ localIP = "127.0.0.1"       │
  │ localPort = 22              │
  │ remotePort = 6022           │
  └─────────────────────────────┘
              [上一步]  [保存并启动]
```

## 六、服务端脚本设计（frps-deploy）

### 脚本功能覆盖

| 脚本 | 功能 |
|------|------|
| `install.sh` | 自动检测系统架构、下载对应 FRP 版本、生成配置、注册 systemd 服务 |
| `uninstall.sh` | 停止服务、清理文件、移除系统服务注册 |
| `update.sh` | 检测最新版本、备份配置、平滑升级 |
| `frps.toml.tpl` | 包含常用配置项注释的模板，方便手动调整 |

### install.sh 核心流程

```
1. 检测 OS 类型（Ubuntu / CentOS / Debian）
2. 检测 CPU 架构（amd64 / arm64）
3. 从 GitHub Releases 下载对应版本
4. 交互式输入：监听端口、Token、Dashboard 端口
5. 写入 /etc/frp/frps.toml
6. 注册并启动 systemd 服务
7. 输出节点信息供 GUI 填写
```

## 七、本地配置校验规则

| 校验项 | 规则 | 级别 |
|--------|------|------|
| TOML 语法 | 格式合法可解析 | 错误 |
| bindPort | 1024 ~ 65535 范围内 | 错误 |
| auth.token | 不为空 | 错误 |
| auth.token | 建议长度 ≥ 16 位 | 警告 |
| webServer.port | 与 bindPort 不冲突 | 错误 |
| 字段类型 | 端口为整数、字符串加引号 | 错误 |

## 八、技术选型

### 最终确认方案：Electron + Vue3

| 层级 | 技术选型 |
|------|----------|
| UI 框架 | Vue3 + TypeScript |
| UI 组件库 | Ant Design Vue |
| 原生层 | Node.js（Electron） |
| 进程管理 | Node.js `child_process` |
| TOML 解析 | `@iarna/toml` |
| 本地存储 | SQLite（via `better-sqlite3`） |
| 网络检测 | Node.js `net.Socket` TCP 探测 |
| 日志处理 | 实时读取 frpc stdout |

### 方案选型过程参考

| 方案 | 框架 | 包体 | 内存 | 开发效率 | 适合场景 |
|------|------|------|------|----------|----------|
| 方案一 | Tauri + Vue3 | ~10MB | 低 | 中 | 有 Rust 经验，追求轻量 |
| **方案二（已选）** | **Electron + Vue3** | **~120MB** | **中** | **高** | **前端团队，优先效率** |
| 方案三 | Electron + React | ~120MB | 中 | 高 | React 技术积累团队 |
| 方案四 | Flutter Desktop | ~30MB | 中低 | 中 | 有移动端扩展计划 |

## 十一、用户核心操作路径


| 01 | 服务端管理方式 | frps 通过 shell 脚本交付，GUI 专注 frpc 客户端管理，两者解耦 | 兼容性更好，职责清晰 |
